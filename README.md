# D&D Silent Master Bot

Telegram-бот для Dungeon Master'а, который позволяет отправлять скрытые события конкретным игрокам во время сессии D&D 5e.

---

## Зачем это нужно

Во время игры в D&D Мастер часто хочет сообщить что-то только одному игроку — например, что его персонаж заметил что-то подозрительное, или что факел начинает гаснуть. В реальной жизни это делается шёпотом. В онлайн-игре — этот бот.

---

## Структура проекта

```
dnd/
├── bot.py              # Точка входа. Запускает бота.
├── handlers.py         # Вся логика обработки команд и кнопок.
├── database.py         # Работа с базой данных SQLite.
├── dnd_api.py          # Интеграция с D&D 5e API.
├── character_sheet.py  # Форматирование листа персонажа.
├── keyboards.py        # Все inline-клавиатуры бота.
├── requirements.txt    # Зависимости Python.
├── .env.example        # Пример файла с токеном.
└── database.db         # База данных (создаётся автоматически).
```

## Описание файлов

### bot.py
Главный файл запуска. Здесь происходит:
- Загрузка токена из `.env`
- Инициализация базы данных
- Создание объекта бота и диспетчера
- Подключение роутера из `handlers.py`
- Запуск polling (бот начинает слушать сообщения)

Если нужен прокси, добавь `PROXY_URL` в `.env` — бот подхватит его автоматически.

---

### handlers.py
Мозг бота. Содержит все обработчики команд и нажатий на кнопки:

- `/start` — приветствие и главное меню
- `/hero` — показать лист персонажа
- `/cancel` — отменить текущее действие
- Нажатия на кнопки: выбор расы, класса, редактирование характеристик, удаление персонажа

Использует FSM (машину состояний) для редактирования полей персонажа — бот запоминает, что именно ты редактируешь, и ждёт ввода.

---

### database.py
Вся работа с SQLite базой данных. Содержит функции для:

- Создания и получения пользователей
- Создания, чтения, обновления и удаления персонажей
- Хранения особенностей персонажа (расовые и классовые черты)

Таблицы в базе:
- `users` — зарегистрированные пользователи и их роли
- `characters` — листы персонажей со всеми характеристиками
- `rooms` — игровые комнаты (для Этапа 3)
- `character_features` — особенности персонажа (темное зрение, боевой стиль и т.д.)

---

### dnd_api.py
Интеграция с открытым [D&D 5e API](https://www.dnd5eapi.co). Загружает:

- Список всех рас и классов
- Подробную информацию о расе (скорость, бонусы)
- Особенности расы (например: Темное зрение, Транс у эльфов)
- Особенности класса на выбранном уровне (например: Второе дыхание у воина)

Если API недоступен (нет интернета или сайт лежит), файл автоматически переключается на встроенные локальные данные — бот продолжит работать.

---

### character_sheet.py
Отвечает за отображение листа персонажа. Содержит:

- `get_modifier(stat)` — вычисляет модификатор характеристики по правилам D&D (например, Сила 14 → +2)
- `format_modifier(value)` — форматирует модификатор со знаком (+2, -1)
- `format_character_sheet(char)` — собирает полный текст листа персонажа
- `format_character_with_features(char, features)` — добавляет к листу расовые и классовые особенности
- `format_character_short(char)` — краткая карточка для списков

---

### keyboards.py
Все inline-клавиатуры бота. Каждая функция возвращает готовую клавиатуру:

- `get_main_menu()` — главное меню (Мой герой / Присоединиться / Стать Мастером)
- `get_character_menu()` — меню персонажа (Редактировать / Удалить / Назад)
- `get_sheet_edit_menu()` — полная сетка редактирования всех параметров
- `get_race_selection_menu(races)` — динамическая сетка рас из API
- `get_class_selection_menu(classes)` — динамическая сетка классов из API
- `get_confirm_delete()` — подтверждение удаления персонажа

---

### requirements.txt
Список зависимостей:

- `aiogram` — асинхронный фреймворк для Telegram ботов
- `aiosqlite` — асинхронная работа с SQLite
- `python-dotenv` — загрузка переменных из `.env` файла

---

### .env.example
Шаблон для файла с секретными данными. Скопируй его в `.env` и заполни:

```
BOT_TOKEN=твой_токен_от_BotFather
PROXY_URL=http://адрес:порт  # необязательно, если Telegram заблокирован
```

Файл `.env` добавлен в `.gitignore` — токен не попадёт в репозиторий.

---

## Установка и запуск

1. Установи зависимости:
```bash
pip install -r requirements.txt
```

2. Создай `.env` из шаблона:
```bash
copy .env.example .env
```

3. Вставь токен от @BotFather в `.env`

4. Запусти бота:
```bash
python bot.py
```
